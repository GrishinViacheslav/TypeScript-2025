  1. Задача 1.

    Описати тип квиток (куди, ціна, піб пасажира, дата). Створити функції для перевірки цього типу (Type Guard, Assert)

  2. Задача 2.

    Обробка Фінансових Транзакцій Клас: Transaction (Транзакція). Містить поля: id, amount, date, type, description. Містить метод для валідації суми. Enum: TransactionType (ТипТранзакції). Визначає напрямок або вид транзакції: DEPOSIT (Поповнення), WITHDRAWAL (Зняття), TRANSFER (Переказ).

  3. Задача 3.

    Конфігуратор бази даних Інтерфейс: IDatabaseConfig – визначає необхідні параметри підключення (host: string, port: number, username: string, password?: string). Клас: DBConnection – містить приватне поле config: IDatabaseConfig і метод connect(). Клас використовує конфігурацію, тип якої задано інтерфейсом.

  4. Задача 4.

    Менеджер стану користувача Клас: UserManager Складний тип: Визначення типу UserStatus. type UserStatus = { state: 'online', lastActive: Date } | { state: 'offline', lastActive: Date } | { state: 'away' }; (Об'єднання для різних станів) Метод: setUserStatus(id: number, status: UserStatus)

  5. Задача 5.

    Валідація величин без створення екземпляра.Статичний метод дозволяє перевіряти дані перед створенням об’єкта.
